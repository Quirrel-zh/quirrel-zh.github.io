Overlay Network 搭建记录

第一步需要购买海外的 VPS。我这里是从 VMiss 购买了一台 CN.HK.BGP.DC2.Basic 搭建 HK 节点（日常使用 + Netflix），一台 JP.TKY.IIJ.Basic 主要用于 AI 解锁等屏蔽中国大陆、香港和台湾省的服务。

这两台 VPS 默认都装载 Debian 13 系统。接下来就可以开始搭建「加速服务」了。

在开启服务器后，使用 Termius 分别通过 SSH 和 SFTP 连接两台服务器，然后更新系统并开启 BBR 加速。

DNS 与域名

准备工作需要在 Cloudflare 中为子域名添加 A 记录，IP 指向对应的服务器。由于有两台 VPS，可创建两个不同子域名，分别指向各自的 IP，方便后续在不同 VPS 中配置。

安装方式概览

下面提供两种方式：一是基于一键脚本自动执行，二是使用 Web UI 手动配置，按需选择。

安装方式一：mack-a 八合一脚本

  1. 安装 mack-a/v2ray-agent 脚本：

安装后可通过 vasma 快捷指令再次打开脚本。

脚本教程：v2ray-agent 文档

按提示依次选择安装 → sing-box。脚本会下载所需组件并初始化 Nginx、申请证书；此时需填写要绑定的域名，建议使用子域名。

绑定域名后，若询问是否使用 DNS API 申请证书，选择 Cloudflare，并按教程创建 CF_TOKEN 并填入。

若使用子域名，脚本可能询问是否用 *.xxxx.com 通过 API 申请通配符证书，选择 yes 可为所有子域名申请证书。

接下来会自动配置各协议，端口、UUID 等可一路回车使用默认或随机值。配置完成后会列出各协议的账号。

在另一台 JP VPS 上同样执行一遍，生成对应协议账号。

再次执行 vasma，选择 7：用户管理 → 查看订阅，可得到自动生成的订阅链接，按需导入到加速工具即可。

此时订阅里多为脚本生成的各协议节点，若需规则分流，可使用 acl4ssr-sub：把脚本里生成的各协议链接逐行复制到文本（两台服务器可各选一部分，视使用习惯而定），粘贴到网页，选择规则（例如 ACL4SSR_Online_Full），生成带规则的订阅链接。

安装方式二：Web 面板

1. 3X-UI（Xray）

3X-UI 是基于网页的开源面板，用于管理 Xray-core。注意： 无法使用 sing-box 核心，只能使用 Xray。

安装时会提示面板端口，可自选或随机。第二步选择是否安装证书，选 Let’s Encrypt for Domain（90 天有效期、自动续期，需开放 80 端口）。

配置成功后输入已在 Cloudflare 中添加的子域名，面板会申请证书；完成后可选择重启服务，并显示公钥、私钥路径（若忘记，后续仍可查看）。同时会显示面板 URL、端口、账号、密码与安全路径。先用 http 打开面板，输入默认用户名与密码，在 面板设置 中填写证书公钥、私钥路径，保存并重启后即可用 https 访问。

在终端中选择 19 SSL 证书设置 → 4 查看存在的域名，可查看当前域名证书公钥、私钥的文件位置。

在 Web UI 的 入站列表 中添加入站，选择协议与用户等；配置完成后，不同用户可通过不同订阅链接订阅节点。

2. s-ui（sing-box）

s-ui 基于 sing-box 核心，提供 Web 管理界面，逻辑与上面类似，可选协议更多。
