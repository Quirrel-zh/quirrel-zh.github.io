如何在 Cursor 中使用高级模型

由于 claude、gpt、gemini 等模型不提供中国区域的服务，导致在 Cursor 中也无法直接使用。即使在设置中一直选中 auto，将 auto/composer 的额度用光后继续使用，虽然显示消耗的是高级模型一栏的额度，但实际上依旧不会调用这些高级模型。

1. 代理软件准备

首先必须通过代理才能访问这些模型服务。

Windows 平台常用软件：

  • Clash for Windows（已停止维护）
  • Clash Verge
  • V2ray

macOS 平台除以上软件外，还有：

  • Loon
  • Stash
  • Surge
  • Proxifier

每个代理软件所监听的默认端口都不同，下文以 Surge 为例进行说明。

2. 代理节点配置

首先需要准备一个可用的机场订阅（节点），将其导入到上述代理软件中。

注意上图红框圈出的部分，分别对应节点、端口号以及增强模式，有以下几点需要注意：

  1. 节点选择：要选择香港/台湾以外的地区，如美国、日本、新加坡。建议优先选择日本或新加坡，延迟相对较低。
  2. 开启 TUN 模式：需要启用 TUN 模式（即虚拟网卡模式），在 Surge 中对应增强模式，可以让电脑中所有流量均经由代理转发。
  3. 记录端口号：需要知道代理软件所监听的 HTTP 端口，例如本例中 Surge 使用的是 6152，Clash 系软件一般为 7890。

3. Cursor 代理设置

在 Cursor 中依次打开 首选项 -> 设置，在上方搜索框中搜索 proxy，在对应选项中填入 http://127.0.0.1:6152（端口号替换为你代理软件实际监听的端口）。

然后在模型选项中关闭 auto，点击你想使用的高级模型，如 Opus 4.6、GPT 5.3 Codex 等。更多模型可以在 Cursor 设置的 Models 一栏中启用或关闭。

完成后再与模型对话，就不会出现模型不可用的提示了。

4. 代理规则（可选）

如果按照上述步骤设置后仍然不可用，通常有两个原因：

  1. 节点不干净：尝试更换其他节点。
  2. 缺少代理规则：需要在代理软件中手动添加 Cursor 相关域名的规则。

5. 不开启全局/TUN 模式的替代方案（仅 Windows）

如果不想开启全局代理或 TUN 模式，可以使用以下工具：

antigravity-proxy

该工具虽然是为「反重力」编写的，但由于反重力与 Cursor 同样使用 Open VSX，因此只需将教程中所有涉及反重力的路径替换为你 Cursor 的安装路径，即可在不开启 TUN 模式的情况下正常使用高级模型。

注意：普通代理本身仍然是必须的，此方案只是省去了开启 TUN/全局模式的步骤。
